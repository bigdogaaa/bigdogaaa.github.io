# 概览

**Obsidian 是你写笔记的“编辑器 + 知识库”，Quartz 是把 Obsidian 笔记变成漂亮网站的“转换器 + 展示层”，Git 是让笔记和网站保持同步、版本控制、自动发布的“运输 + 备份工具”。**

它们通常一起组成一个非常流行的免费“个人数字花园 / 博客”方案，尤其是配合 **GitHub Pages** 使用。

### 三者的定位与关系图

```
你 ↔ Obsidian（写作 & 管理笔记）
          ↓
       笔记文件（.md + 图片）
          ↓
       Git（版本控制 + 同步）
          ↓
       Quartz（静态网站生成器）
          ↓
GitHub / GitHub Pages（免费托管 + 自动构建）
          ↓
       全世界都能看到的个人网站
       (https://你的用户名.github.io)
```

### 详细说明每个工具的作用和它们如何协作

| 工具       | 是什么？                                   | 主要作用                                   | 和另外两个的关系                                   | 是否必须有？ |
|------------|--------------------------------------------|--------------------------------------------|----------------------------------------------------|--------------|
| **Obsidian** | 本地 Markdown 笔记软件（双向链接、神器级） | 写笔记、整理知识、画思维导图、日常使用     | 产生 .md 文件 → 交给 Git 管理 → 交给 Quartz 转换 | 核心（你写东西的地方） |
| **Quartz**   | 免费、开源的静态网站生成工具（基于 Obsidian 语法） | 把一堆 .md 文件变成现代、美观的网站<br>（支持双链、图谱、标签页、搜索、深色模式、Mermaid 等） | 读取 Obsidian 风格的 Markdown → 生成 HTML/CSS/JS | 核心（决定网站长什么样） |
| **Git**      | 版本控制系统（分布式）                     | 跟踪文件变化、备份、多设备同步、推送远程   | 管理 Obsidian 笔记文件夹 + Quartz 配置<br>把改动推到 GitHub | 几乎必须（没有 Git 很难自动部署） |

### 最常见的实际协作流程（2026 年主流玩法）

1. 你在 **Obsidian** 里写笔记（用 `[[链接]]`、`#标签`、callout、图片等）
2. 使用 **Obsidian Git** 插件（或手动）把笔记文件夹 commit + push 到 GitHub
3. **Quartz** 放在同一个仓库里（或者通过 Actions 拉取你的笔记）
   - Quartz 会读取 `content/` 文件夹里的 .md 文件
   - 把它们转换成静态网页（保留 Obsidian 大部分语法）
4. **GitHub Actions** 自动运行 Quartz 的构建命令
5. 构建出来的网页被 GitHub Pages 托管
6. 你打开浏览器 → https://你的用户名.github.io 就能看到最新笔记了

### 几种常见的仓库组织方式

| 方式                        | 仓库结构简述                              | 适合人群                     | 优缺点简评                              |
|-----------------------------|-------------------------------------------|------------------------------|-----------------------------------------|
| **方式1：一个仓库全放一起**（最常见） | 仓库里既有 Quartz 文件，也有 content/ 放笔记 | 新手、单人使用               | 简单，但私人笔记会暴露（除非设 private + 选择性发布） |
| **方式2：两个仓库**         | 仓库A：Obsidian vault（可 private）<br>仓库B：Quartz + GitHub Actions 拉取 A 的内容 | 想把私人笔记和公开内容分开   | 更安全，但设置稍复杂                    |
| **方式3：Quartz 作为子目录** | Obsidian vault 里新建 quartz/ 子文件夹放 Quartz | 喜欢一切都在一个 vault 里    | 方便，但容易冲突                        |

### 总结一句话关系

- **Obsidian**：你日常写和整理的地方
- **Quartz**：把你写的笔记变成好看网站的工具
- **Git**（+ GitHub）：把笔记和 Quartz 配置同步到云端，并让 GitHub 自动把它们变成在线网站

这三者缺一不可地组成了目前最流行、最强大的**免费 Obsidian → 个人网站**方案之一。

# 搭建过程
Quartz 的安装和部署到 GitHub Pages 其实非常直观，目前（2026年2月）官方推荐的方式非常简洁。下面给你最清晰的**一步一步操作流程**，适合零基础或刚开始的用户。

### 前置条件（必须先准备好）
- **Node.js**：**至少 v22**（推荐最新 LTS 版本）
  - 下载地址：https://nodejs.org/
  - 安装后在终端运行 `node -v` 确认版本 ≥ 22.x
- **Git**：已安装并能正常使用（大多数电脑自带或之前装过）
- 一个 **GitHub** 账号

### 推荐安装方式（最简单、最常用）

#### 步骤 1：在本地创建 Quartz 项目

1. 打开**终端**（Windows 用 PowerShell 或 Git Bash，Mac 用 Terminal）
2. 进入你想存放网站的文件夹，例如：

   ```bash
   cd ~/Documents
   # 或者 cd D:\Projects 等等，随便你喜欢放哪里
   ```

3. 依次运行以下命令（一行一行复制粘贴执行）：

   ```bash
   # 1. 克隆 Quartz 仓库
   git clone https://github.com/jackyzha0/quartz.git my-quartz-site
   # 你也可以把 my-quartz-site 改成你喜欢的名字

   # 2. 进入目录
   cd my-quartz-site

   # 3. 安装依赖
   npm i

   # 4. 初始化（最重要一步，会问你几个问题）
   npx quartz create
   ```

   在 `npx quartz create` 这一步，程序会问你几个问题，建议这样选：

   - **Content folder**：选 `content`（默认即可）
   - **Whether to initialize with an example**：选 **Yes**（先看看示例效果，后面可以删）
   - **Bundle version**：选 **latest** 或默认

   完成后你会看到一个基本的 Quartz 站点结构。

#### 步骤 2：把你的 Obsidian 笔记放进去

最常见两种做法，任选一种：

**方式 A（推荐新手）**：直接把 Obsidian vault 复制过来

1. 找到你的 Obsidian 仓库文件夹（vault）
2. 把整个 vault 里的内容**复制**到 `my-quartz-site/content/` 文件夹中（可以覆盖掉里面的示例文件）
   - 建议先备份原 vault

**方式 B（推荐长期使用）**：保持 Obsidian vault 原位置，使用**软链接**或 **Obsidian Git 插件**同步

- 以后再详细展开，初期建议先用方式 A 快速看到效果

#### 步骤 3：本地预览效果（强烈建议先看一眼）

```bash
# 在 quartz 根目录运行
npx quartz build && npx quartz serve
```

然后浏览器打开 http://localhost:8080  
就能看到你的网站雏形了！（改动 md 文件后会自动刷新）

#### 步骤 4：推送到 GitHub 并开启 GitHub Pages

1. **在 GitHub 上新建一个仓库**
   - 仓库名建议：`你的用户名.github.io`（这样网址就是 https://你的用户名.github.io）
   - **不要**勾选 Initialize with README、.gitignore 等

2. 把本地 Quartz 关联到这个远程仓库

   在终端（还在 quartz 目录里）运行：

   ```bash
   # 把下面这行 YOUR-USERNAME 和 REPO-NAME 改成你自己的
   git remote add origin https://github.com/YOUR-USERNAME/REPO-NAME.git

   # 或者直接复制 GitHub 给你的 HTTPS 地址
   ...
   # 强制覆盖远程分支
   git push --force origin main
   ```

3. 第一次推送全部内容

   ```bash
   npx quartz sync
   ```

   （这个命令会 build + commit + push，非常方便）

4. **开启 GitHub Pages**

   - 进入你的 GitHub 仓库 → **Settings** → **Pages**
   - Source 选 **GitHub Actions**
   - 保存

5. **添加 GitHub Actions 自动部署**（非常重要！）

   在 quartz 项目根目录创建文件夹和文件：

   ```
   quartz/.github/workflows/deploy.yml
   ```

   把下面内容粘贴进去（这是官方最新推荐的 workflow，2026年2月版本）：

   ```yaml
   name: Deploy Quartz site to GitHub Pages
   
   on:
     push:
       branches:
         - main
   
   permissions:
     contents: read
     pages: write
     id-token: write
   
   jobs:
     build:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
           with:
             fetch-depth: 0  # 获取全部历史，用于git log统计等功能
         - name: Setup Node
           uses: actions/setup-node@v4
           with:
             node-version: 22
         - name: Install Dependencies
           run: npm ci
         - name: Build Quartz
           run: npx quartz build
         - name: Upload artifact
           uses: actions/upload-pages-artifact@v3
           with:
             path: public
   
     deploy:
       needs: build
       runs-on: ubuntu-latest
       environment:
         name: github-pages
         url: ${{ steps.deployment.outputs.page_url }}
       steps:
         - name: Deploy to GitHub Pages
           id: deployment
           uses: actions/deploy-pages@v4
   ```

6. 再次运行：

   ```bash
   npx quartz sync
   ```

   推送后，GitHub 会自动构建，1-5 分钟后访问：

   https://你的用户名.github.io

   就看到你的网站啦！

### 常用后续命令速查

```bash
# 本地预览
npx quartz serve

# 只构建（不启动服务器）
npx quartz build

# 构建 + 提交 + 推送（最常用）
npx quartz sync

# 更新 Quartz 到最新版
npx quartz update
```

### 常见问题快速解决

- 构建报错 Node 版本不对 → 升级到 22 或更高
- 图片不显示 → 确保图片放在 `content/` 里，路径用相对路径
- GitHub Pages 没反应 → 确认 Pages 设置里 Source 是 **GitHub Actions**
- 想改网站标题、图标、布局 → 编辑 `quartz.config.ts` 文件
